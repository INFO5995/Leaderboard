Privacy-Sensitive Disclosure Logic

For disclosures where technical details are withheld, the leaderboard uses privately held evidence and vendor confirmation to score impact without publishing exploit steps, affected endpoints, or examples of exposed personal data.

  • Severity High is used when private evidence shows unauthorized access to sensitive personal information for other users. Critical is reserved for broad, easily repeatable exposure at substantial user-base scale, or where the receiving platform or vendor classifies it as Critical.
  • Impact The impact evidence score is 2 when the affected data class, access path, vendor confirmation, and remediation evidence are documented privately.
  • Novelty The novelty score is 2 when the disclosure is a confirmed zero-day: no prior public report or known duplicate is available, and the affected organization confirmed and fixed the issue.
  • Display Public records remain intentionally generic: the page can show confirmation, fix status, severity, and score logic, but not sensitive data types, PoCs, logs, or implementation details.

Display Notes

Zero-day status is shown from the data file when explicitly provided. If it is not provided, the page infers it from the novelty score where possible: 2 is treated as zero-day, 1 as non-zero-day, and unsupported values as unknown.

The records table shows one severity value. Platform or vendor severity is preferred when available; otherwise the page marks the value as internally assessed and uses the documented scoring rationale.
